# 自定义模块
+ 一个模块本质上就是一个py文件
+ 自己定义一个模块，其实就是自己写一个py文件
+ 如果一个py文件想要当作一个模块导入，文件名一定要遵守命名规范：由数字、字母、下划线组成，但不能以数字开头
+ 导入模块后，就能使用这个模块里的变量和函数

```
import my_module

from my_module import *

'''
# 两种导入方法的区别：
  - import my_module 方法将my_module导入，在调用时许采用 my_module.fuction()的模式调用my_module中的函数或变量
  - from my_module import * 方法是将模块my_module中的函数和变量导入，并未导入my_module，在使用时直接采用function()形式使用函数即可，不再需要写模块名。
  
'''
```

+ 使用from <module_name> import * 是导入这个模块中“所有”的变量和函数；
  - 其中的“所有”并非全部
  - 其本质上是读取模块里的 __all__ 属性，调入的内容要看这个属性里定义了哪些变量和函数
  - 如果模块里没有定义 __all__ 才会导入模块内所有不以 _ 开头的函数和变量
  - 以一个下划线开始的变量，建议只在本模块中使用，别的模块不要导入
    + 该模式的变量在使用from <module_name> import * 这种方式无法调用
  
```
__all__ = ['m','test']     # __all__ 为一列表，列表内包含的内容才会被导入，不在列表内的内容不会被导入

```

+ __name__: 当直接运行py文件时，其值为 __main__
            当py文件作为一个模块被导入时，其值为该py文件的文件名
